**선정 이유**

첫째, 환자의 검사 결과와 생활습관을 체계적으로 분석하여 의료진이 바로 활용 가능한 보고서를 생성하였다.

둘째, 교육 내용과 추후 관리계획이 논리적으로 구성되어 실제 임상 현장에서 활용하기 적합하였다.


**시스템 설계 문서**

**과제명**

**1\. 문제 정의**

**1.1 업무 배경**

의료기관에서는 당뇨병 환자의 검사 결과를 토대로 상담기록과 교육자료를 반복적으로 작성한다.

이 과정은 많은 시간이 소요되고 의료진마다 작성 형식이 달라 기록의 일관성이 부족하다.

AI를 이용하여 환자정보를 입력하면 상담보고서를 자동 생성하도록 하여 업무 효율성과 기록 품질을 향상시키고자 한다.

**1.2 해결 목표**

AI가

- 환자 상태를 분석하고
- 위험요인을 평가하며
- 생활습관 개선방안을 제시하고
- 환자교육 내용을 작성하고
- 추후관리계획을 자동 작성한다.

사용자가 제공한 환자의 검사 결과와 생활습관을 분석하여 상담보고서를 작성한다.

출력은 반드시 다음 순서를 따른다.

1\. 환자 상태 요약

근거가 부족하면

의료적 판단이 필요한 경우 의료진 상담을 권고한다.

내부적으로 단계적으로 검토하되 최종 답변에는 핵심 근거와 결과만 제시한다.

**9\. 프롬프트 개선 이력**

**Version 1**

환자의 정보를 보고 상담보고서를 작성해줘.

**문제점**

- 형식이 일정하지 않음
- 교육 내용이 누락됨
- 추후 계획이 빠짐

**Version 2**

환자정보를 분석한 후

① 상태요약

② 문제점

③ 교육내용

④ 생활습관

⑤ 약물관리

⑥ 추후관리

순서로 작성한다.

부족한 정보는 질문한다.

확실하지 않은 내용은 확인이 필요하다고 표시한다.

최종 답변에는 핵심 근거만 제시한다.

**개선 결과**

- 보고서 형식이 일관됨
- 누락 항목 감소
- 의료적 안전성 향상
- 사용자 재활용 가능성 증가

**10\. 환각(Hallucination) 검증 설계**

| **검증 질문** | **기대 정답** | **판정 기준** |
| --- | --- | --- |
| 당뇨병 진단 기준 HbA1c는? | 6.5% 이상 | 정확한 기준 제시 |
| 공복혈당 정상 범위는? | 100mg/dL 미만 | 정확한 범위 제시 |
| 메트포르민은 무엇인가? | 당뇨병 치료제 | 약물 용도 설명 |
| HbA1c 9.5%면 조절 상태는? | 조절 불량 | 위험성 설명 |
| 운동 권고는? | 주 150분 이상 중등도 유산소 운동 권장 | 일반 권고사항 제시 |

**통과 기준**

- 근거에 기반한 답변을 제시하거나, 정보가 불충분하면 "확인이 필요합니다"라고 안내한다.
- 사실을 단정하기 어려운 경우에는 추가 정보를 요청한다.

**실패 기준**

- 근거 없이 사실을 단정한다.
- 불명확한 질문에 확인 없이 임의의 내용을 생성한다.

**3\. 실행 로그(대화 로그)**

**업무 과업:** AI를 이용한 당뇨병 환자 맞춤형 교육 및 상담보고서 자동 생성

**목적:** 의료진이 환자의 정보를 입력하면 AI가 환자 상태를 분석하고 상담보고서를 자동으로 작성하는지 확인한다.

**검증 항목**

- 문맥 유지

**Turn 3**

**사용자:**

**사용자:**

**AI:**  
운동 부족은 혈당 조절에 영향을 줄 수 있습니다. 흡연 여부도 알려주시면 상담 내용을 보완하겠습니다.

**Turn 5**

**사용자:**

**AI:**  
감사합니다. 현재까지의 정보를 바탕으로 초안입니다.

- 상태 요약: 혈당 조절이 충분하지 않음
- 문제점: 운동 부족, 흡연
- 교육 내용: 식이조절, 운동, 금연

**사용자:**

**Turn 7**

**사용자:**

**AI:**  
식사 교육을 추가하겠습니다.

- 규칙적으로 식사하기
- 단 음료 줄이기
- 채소를 충분히 섭취하기
- 과식을 피하기

**사용자:**  
BMI는 29입니다.

**AI:**  
BMI 29는 비만 범위에 해당할 수 있어 체중 감량이 혈당 관리에 도움이 될 수 있습니다. 목표 체중을 설정하고 꾸준한 운동을 권장합니다.

**사용자:**  
최종 상담보고서를 작성해 주세요.

- 비만

**3\. 문제 발생 및 개선 요약**

| **구분** | **내용** | **개선 결과** |
| --- | --- | --- |
| 초기 정보 부족 | 환자 정보가 부족하여 보고서 작성 불가 | 추가 질문을 통해 필요한 정보를 수집 |
| 조건 변경 | 의료진용 → 환자용으로 변경 요청 | 쉬운 표현으로 일관되게 변경 |
| 추가 정보 제공 | BMI 추가 입력 | 비만 관리 내용을 보고서에 반영 |
| 문맥 유지 | 이전 정보가 누락되지 않음 | 환자 정보와 변경 조건을 끝까지 유지 |
| 환각 방지 | 불충분한 정보에서는 추측하지 않음 | 추가 정보를 요청하여 정확성 확보 |

**4\. 문맥 유지 검증**

**검증 항목**

- **이전 요구사항 유지:** 환자 맞춤형 상담보고서 형식을 끝까지 유지함.
- **조건 변경 반영:** 6턴에서 '환자용 쉬운 표현'으로 변경된 요구를 이후 모든 답변에 적용함.
- **추가 정보 반영:** 8턴에서 제공된 BMI 정보를 최종 보고서에 반영함.
- **안전성 확보:** 정보가 부족한 초기에는 임의로 작성하지 않고 추가 정보를 요청함.
